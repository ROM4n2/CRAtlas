# app/not-found/

自定义 404 页面——当用户访问不存在的路由时显示。

## 职责

当用户访问未匹配的 URL 时，Next.js 自动渲染此页面，提供友好的中文提示与返回首页的链接，替代 Next.js 默认的 404 页面。

## 文件清单

|------|------|

## 使用方式

`not-found.tsx` 是 Next.js App Router 的特殊约定文件，放置在 `app/` 目录下时，框架自动在路由未匹配时渲染此组件，无需手动路由注册。

约定要点：
- 必须是 `export default` 的客户端组件
- 文件名固定为 `not-found.tsx`
- 仅处理 404（Not Found），不处理 500 等其它错误

## 约定

- **无状态**：纯展示组件，不依赖任何全局状态或数据查询
- **纯静态**：适合 SSG 导出，无客户端副作用
- **样式**：使用 Tailwind CSS 工具类，与全站风格一致
